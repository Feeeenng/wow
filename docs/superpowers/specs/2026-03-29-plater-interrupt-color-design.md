# Plater 打断染色模组 设计文档

**日期：** 2026-03-29
**目标：** 为魔兽世界12.0大秘境开发一个 Plater 模组，根据怪物需要的打断人数对血条进行静态染色。

---

## 背景

5人大秘境中，不同怪物对打断的优先级不同：有的只需1人打断，有的需要2人协调，有的需要3人全力打断。现有工具（如 Kickit+）专注于打断冷却监控，但没有对怪物本身做视觉分级。本模组通过血条颜色直接传达"这个怪需要几人打断"，减少沟通成本。

---

## 功能范围

- 根据 NPC ID 查表，对血条进行静态染色
- 支持 1人 / 2人 / 3人 三个等级，每级一个可配置颜色
- 不在列表里的怪物保持 Plater 原有颜色，不干扰
- 提供 Plater 配置界面调整颜色，无需改代码
- NPC 数据表在脚本顶部维护，格式极简

```

```

## 数据结构

脚本顶部维护 NPC 数据表，用户手动添加：

```lua
-- ============================================================
-- 打断分组数据表
-- 格式：[NPC_ID] = 需要几人打断 (1 / 2 / 3)
-- 添加新怪物只需在这里加一行
-- ============================================================
local INTERRUPT_LIST = {
    -- [12345] = 1,  -- 怪物名（副本名）
    -- [67890] = 2,
    -- [11111] = 3,
}
```

---

## 配置项

在 Plater 配置界面展示，默认展开可见：

| key | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| `enabled` | toggle | true | 启用/禁用整个模组 |
| `color1` | color | `{0, 1, 0, 1}` 绿色 | 1人打断的血条颜色 |
| `color2` | color | `{1, 0.5, 0, 1}` 橙色 | 2人打断的血条颜色 |
| `color3` | color | `{0.2, 0.6, 1, 1}` 蓝色 | 3人打断的血条颜色 |

配色逻辑：绿→橙→蓝 视觉区分明显，蓝色代表高威胁需全力打断，与大秘境常用 UI 色彩语义一致（避开已用的黄色、红色、紫色）。

---

## Hook 逻辑

挂载 Plater 的 `OnShow` 和 `OnUpdate` 事件：

```
名条显示/更新
  ↓
从 UnitGUID 解析 NPC ID
  ↓
查 INTERRUPT_LIST[npcID]
  ↓ 命中              ↓ 未命中
取 color1/2/3        不处理，保持原色
  ↓
调用 Plater SetHealthbarColor 染色
```

关键实现细节：
- NPC ID 从 GUID 第六段解析：`tonumber(select(6, strsplit("-", UnitGUID("nameplate"))))`
- 颜色从模组配置读取：`modConfig["color" .. count]`
- `enabled` 为 false 时跳过所有处理

---

## 不在范围内

- 动态检测实际打断就绪人数（不需要）
- 施法条染色（只做血条）
- 自动预填 NPC 数据（数据由用户自己维护）
- 多语言支持（只做中文注释）
